# SPEC §5 — K-Space as Address Space ("Transactional Tape" Test)

**Dated:** 2026-07-17
**Status:** pre-registered, not yet run
**Models:** the trio — `gpt2` (124M), `EleutherAI/pythia-410m`, `Qwen/Qwen3-0.6B`
**Scale:** weekend. One 24GB GPU (rented) or CPU-with-patience; total compute + API budget target < $40.

---

## 0. Question

Do K vectors in the cache function as **content addresses** — same-referent mentions
clustering in K-space, state revisions committed as new cells at a *similar* K
(append-only delta-writes), with |V| acting as **version dominance** at a contested
address — or is K-geometry dominated by position and token surface form?

Three sub-questions, each with its own measurement:

- **(Write/address)** Do mentions of the same referent produce nearby keys, beyond
  what lexical identity and position predict?
- **(Namespace)** Does rotary position encoding *split* same-referent addresses by
  position (RoPE as automatic version namespacing)?
- **(Version/read)** Across an update chain for one referent, does ‖v‖ / ‖W_O v‖
  grow with update index (later writes out-shout earlier ones), and does a
  retrieving query's α implement latest-wins, averaging, or first-wins?

---

## 1. Why the trio is the right instrument

| | gpt2 | pythia-410m | Qwen3-0.6B |
|---|---|---|---|
| Rotation of K | **none** (learned absolute pos embs, position enters via residual) | **partial**: `rotary_pct=0.25` → first 16 of 64 head dims rotated, 48 static | **full** RoPE, all dims |
| K normalization | none | none | **QK-RMSNorm** per head → ‖k‖ ≈ const, direction-only addressing *by construction* |
| K bias `b_K` | present | present | absent |
| KV heads | 12 (MHA) | 16 (MHA) | **8 KV heads, 16 Q heads (GQA)** |
| Layers × d_head | 12 × 64 | 24 × 64 | 28 × 128 |
| Usable ctx | **1024** (hard ceiling) | 2048 | ≥32k |

**The RoPE dosage gradient (0% → 25% → 100%)** turns the namespace hypothesis
into a dose–response prediction rather than a binary. Pythia additionally gives a
**within-model, within-head control**: compute every purity metric separately on
its static-dim subspace vs rotated-dim subspace. Same weights, same data, same
head — only the rotation differs. No published work has this contrast.

Qwen3's QK-norm answers the standing question from the §2/§4 notes (does bounding
K produce tighter address clusters?) inside the same run.

**Known confound to carry:** GPT-2 keys are *not* position-free — absolute
position embeddings enter K additively via the residual. So "no rotation" ≠ "no
position." §5.4 measurement M5 (position probe) quantifies positional
contamination per model instead of assuming it.

## 2. Corpus (the part worth spending on)

Hard constraint: all cross-model documents ≤ **950 tokens** in the *GPT-2*
tokenizer (leaves room for the final probe question inside ctx 1024). An optional
Qwen3-only long-range extension (§5.8) relaxes this.

Every document carries **token-level gold labels**: `(referent_id, mention_idx,
update_idx, surface_form)` for each mention span, emitted by the generator —
no coref model in the loop.

### Track A — synthetic state-update interleave (primary; free; deterministic)

Generator (own script, seeded): N referents × M state updates each, interleaved,
with filler sentences of controllable length between updates.

- Referent types: (a) code variables `x = 3 … x = 7`; (b) object–location
  narrative "The marble is in the red box … The marble is now in the blue jar";
  (c) key–value facts "Dana's phone number is …".
- Sweep: N ∈ {4, 8, 16}, M ∈ {2, 4, 6}, filler gap ∈ {short ~10 tok, long ~80 tok}.
- **Surface-form split (load-bearing):**
  - `same-surface`: every mention uses the identical token(s) ("marble" … "marble").
  - `diff-surface`: coreferent chain with varying surface ("the marble" → "it" →
    "the glass sphere"), realized from a synonym/pronoun table so gold labels stay exact.
- **Built-in controls, same documents:**
  - `same-type/diff-referent`: two referents sharing a head noun ("the red box"
    vs "the blue box") — separates lexical clustering from referent addressing.
  - `position-matched/diff-referent`: mention pairs at matched token-distance.
- Each document ends with a probe question for one randomly chosen referent
  ("what is x?" / "Where is the marble?") — this powers the read-semantics test
  and a behavioral check (does the model's greedy answer use the *latest* value?).
- Volume: 300 documents (≈ 100 per referent type), balanced over the sweep.

### Track B — code reassignment (free; deterministic)

150 short Python snippets: real-looking functions with 2–4 reassigned variables,
distractor variables never reassigned, final `print(x)` as probe. Generated by
template + AST check. Gives naturalistic *syntax* while keeping gold labels exact.

### Track C — natural narrative (the API spend; optional but recommended)

~150 short stories generated via Claude API with mention spans **emitted as
inline markup by the generator prompt**, then stripped + validated (regex +
manual spot-check of 15). Adds surface diversity Track A can't fake. Est. cost:
low single-digit dollars. If validation is noisy, demote Track C to
"exploratory" and adjudicate all P5.* on Tracks A+B only.

## 3. Extraction

HF transformers, forward hooks; bf16; one doc per forward; no generation except
the final probe answer (greedy, ≤8 tokens).

Per (layer, **KV** head), record for labeled mention tokens + a 10% random
background sample + token 0 (kept separately for sink stats):

1. `k_pre` — output of `k_proj` (+ `b_K` where present); for Qwen3 **post
   QK-RMSNorm, pre-rotation** (norm is part of address computation; also store
   pre-norm raw for one pilot layer to confirm it doesn't matter).
2. `k_post` — the rotated key actually cached (`past_key_values`).
3. `v`, plus `‖v‖₁, ‖v‖₂` and **`‖W_O v‖₂`** (slice `o_proj` per head; for GQA,
   compute under each of the 2 sharing Q-heads and store both).
4. `α` received by every labeled mention token *from the probe-question query
   positions* (realized attention, per Q-head).
5. Residual-stream norm at each labeled token (massive-activation flag, per §survey:
   threshold = 5× layer median).

**Storage discipline (learned from §3 data-volume incident):** labeled +
sampled positions only → ~50–150 MB per model total. Store one npz per
(model, doc). Full-token dumps are pilot-only, one doc per model.

Sanity gates before any analysis (blocks, per §4-hygiene precedent):
- Reconstructed `k_post` from `k_pre` + own RoPE implementation matches cached
  values ≤ 1e-3 rel. err. on a pilot doc, per model. (This is the §4a lesson:
  a check that can't fail is not a check — perturb once to verify it *can* fail.)
- Pythia: confirm from config which 16 dims rotate; verify static dims of
  `k_pre` == static dims of `k_post` exactly.
- Qwen3: confirm hook order `k_proj → k_norm → RoPE` against source; confirm
  8 KV heads and the Q→KV head map.

## 4. Measurements

All similarity = cosine. Primary coordinates: **head-mean-centered** keys
(removes the constant `b_K` offset in gpt2/pythia; harmless for Qwen3); report
raw as secondary. Sink/massive-activation tokens excised from M1–M4, reported
separately in M6.

- **M1. Address purity (write test).** Per (layer, head, key-variant ∈
  {pre, post}): AUC of same-referent-pair vs each control-pair class
  (`same-type/diff-referent`, `position-matched/diff-referent`), from pairwise
  cosine. No clustering algorithm needed — pairwise AUC avoids choosing k.
  A head is an **address head** if AUC > 0.9 against *both* controls.
  (HDBSCAN cluster-purity view as a secondary visualization only.)
- **M2. Lexical vs semantic addressing.** M1 restricted to `diff-surface`
  chains. This is the discriminating condition: purity here can't be lexical echo.
- **M3. Namespace test.** ΔAUC = purity(pre) − purity(post) per head, as a
  function of mention-pair token distance. Cross-model: dosage curve of ΔAUC at
  long range vs rotation fraction (0/0.25/1.0). Within Pythia: purity on static-48
  subspace vs rotated-16 subspace, same pairs, same heads.
- **M4. Version dominance + read semantics.**
  (a) Within each gold update chain: Spearman ρ of update_idx vs {‖v‖₁, ‖v‖₂,
  ‖W_O v‖}; paired across chains, per head; report heads×layers heatmap.
  (b) Probe-time α over the chain's versions, per Q-head: classify
  latest-wins (α_last > 0.6) / averaging (max α < 0.4) / first-wins / other.
  (c) Behavioral: fraction of greedy probe answers using the latest value,
  per model — ties mechanism to behavior.
- **M5. Position contamination covariate.** Ridge regression k → token position,
  per (layer, head, variant); R² reported alongside every M1/M3 number.
  Expected: gpt2 nonzero even for `k_pre` (absolute pos in residual).
- **M7. Effective read support (version-count coupling).** At each probe query,
  per Q-head, compute (a) raw participation ratio n_eff = 1/Σαᵢ² and
  exp(entropy) over non-sink positions; (b) **commit-weighted** n_eff on
  α·‖W_O v‖ renormalized (sinks excised). Report the (a)−(b) gap as abort
  overhead. Primary regression: n_eff vs gold M ∈ {2,4,6} within address heads
  (from M1). Slope ≈ 0 → latest-wins; ≈ 1 → averaging; ≈ const 2 or
  intermediate → delta/overshoot. This is the continuous form of M4b.
  Secondary (Tracks C + background sample): distribution of commit-weighted
  n_eff vs the corpus mention-count distribution — shape match (heavy tail)
  vs per-head constant.
- **M6. Sink ledger.** Token-0 / massive-activation cells: their ‖v‖, ‖W_O v‖,
  received α — the "third cell type" stats, kept out of the main analysis and
  compared against the survey's value-state-drain profile per model. (Qwen3 has
  no b_K route to a sink — if it drains anyway, that feeds the §4 bias-vs-content
  sink discussion directly.)

## 5. Pre-registered predictions

- **(P5.a)** Address heads (M1 AUC > 0.9 vs both controls) exist in all three
  models, concentrated in middle-depth layers, and are a minority of heads.
- **(P5.b)** At least some address heads survive the `diff-surface` restriction
  (M2) in every model → addressing is at least partly semantic, not purely
  lexical. *If purity collapses under M2 everywhere, the address story reduces
  to token-matching (induction-flavored) and the transactional framing is
  weakened at the root.*
- **(P5.c)** Namespace dose–response: long-range ΔAUC(pre−post) increases with
  rotation fraction (gpt2 ≈ 0 < pythia < qwen3), **and** within Pythia,
  static-subspace purity > rotated-subspace purity for long-range pairs.
- **(P5.d)** Version dominance: ‖W_O v‖ increases with update_idx (median ρ > 0
  with CI excluding 0) in retrieval-relevant heads. Genuinely uncertain — the
  §survey flat-‖v‖₂ result (CV ≈ 0.02–0.03) argues against; the ℓ1/ℓ2 and
  raw-vs-projected discrepancies argue it's untested. Either outcome is a result.
- **(P5.e)** Read semantics: probe α is latest-wins, not averaging, in the heads
  that drive correct behavioral answers. If averaging + correct answers co-occur,
  the convex-fold problem is being solved elsewhere (overshoot deltas or MLP) —
  flag for follow-up, don't chase this weekend.
- **(P5.f)** QK-norm tightens addresses: Qwen3 within-referent cosine spread
  (matched relative depth) < gpt2/pythia. Adjudicates the standing §2/§4 question.
- **(P5.g)** Effective read support is a *data* property, not a head property:
  (i) within address heads, commit-weighted n_eff vs M has slope ≈ 0
  (latest-wins) — consistent with P5.e — and specifically does NOT track M with
  slope ≈ 1; (ii) on natural text, commit-weighted n_eff across queries is
  heavy-tailed and tracks per-referent mention counts, refuting the
  per-head-constant model of attention sparsity. *(ii) is the tape hypothesis's
  only prediction that no descriptive sparsity account also makes — highest
  novelty per measurement in the spec.*

## 6. Decision tree

- **P5.a+b hold, P5.c holds** → K is a semantic address space; position acts as
  a namespace, dose-dependently. Consequences: (i) §1-Exp1 |V| eviction is
  reframed as *per-address version eviction* — a global |V| sort would evict
  superseded versions, which is the desired behavior, so revive Exp 1 with
  address-local normalization; (ii) §1-Exp3 ANN sieving: green light on `k_pre`
  clustering, red flag on `k_post` at long range — sieve pre-rotation, re-rotate
  survivors; (iii) §-set-theory paper gains its central positive example.
- **P5.a holds, P5.b fails** → addresses are lexical echoes; transactional tape
  is a metaphor for induction machinery. Still publishable as a negative
  boundary; feeds the set-theory paper's "which sequence assumptions are
  load-bearing" section instead.
- **P5.c fails flat (no dosage effect)** → RoPE does not namespace; position and
  content coexist in K without interference at these scales. ANN sieving safe on
  `k_post` directly (simplifies Exp 3).
- **P5.d holds** → the |V| write-dominance mechanism is real → design the
  training-loop version (sorted retention, §KV-chat "the model learns to shout")
  with actual evidence behind it.
- **P5.d fails + P5.e averaging + behavior correct** → revision is implemented
  by overshoot or outside attention; |V|-as-priority is dead for revision
  tracking (keep for §1-Exp1 only in its original importance framing).
- **P5.g(i) slope ≈ 1 (averaging) while behavior is correct** → the machine keeps
  all versions live and resolves elsewhere → per-address |V| eviction (branch 1's
  reframed Exp 1) would *delete load-bearing history* — kill that reframing.
  Slope ≈ 0 → superseded versions are provably dead weight → strongest possible
  license for version-eviction, and the n_eff-vs-mention-count distribution
  becomes the headline figure.
- **Everything fails** → K-geometry at 124M–0.6B is position+syntax only.
  Honest negative; the trio census (§2–§4) remains the main thread; note that
  the scale caveat (0.6B ceiling) limits the claim.

## 8. Known threats to validity (recorded now, so the REPORT must address them)

1. Scale ceiling: 0.6B max — semantic addressing may emerge later; absence here
   is not absence at 7B. Claims must be scale-qualified.
2. GPT-2's additive position (M5 covariate) can mimic "no namespacing" (position
   hides in both pre and post identically). M5 R² must be cited next to any
   gpt2 namespace claim.
3. Synthetic corpus regularity: templates may induce artificial clustering.
   Track B/C agreement is the check; report per-track, never pooled-only.
4. GQA head sharing (Qwen3): α in M4b lives in Q-heads (16), addresses in
   KV-heads (8). Keep the map explicit in every plot label.
5. Multiple comparisons across (layer × head × variant): report head *counts*
   above threshold vs a shuffled-referent-label null (100 shuffles), not lone
   star heads.
